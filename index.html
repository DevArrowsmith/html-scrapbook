Header

Headers are primarily used for titles.

  • <h1> is used for main titles, such as a page or website title. Only one should be used per page or view.
  • <h2> is used for secondary titles, eg. the section titles in this document, or titles for sections of an article.
  • <h3> onward are used for lower level titles.
  • It is best practice not to skip title levels, in part because this can interfere with screen reader use.
  • Although header elements adjust text size they should not be used for this purpose (unless it is a fortunate side effect of using a header appropriately as a title following the above rules). Instead use styling to modify text size.
  • You can use styling to adjust the size of header text if the default font size is inappropriate.

<h1>Header 1</h1>
<h2>Header 2</h2>
<h3>Header 3</h3>
<h4>Header 4</h4>

Image

  • Image elements display images using a link to an image file.
  • The image file can be local or remote.
  • Self-closing.
  • The src attribute is the link to the image. If the image is local the route needs to be written relative to the location of the file that's accessing the image. If the image is non-local (on another website) a regular URL works.
  • The alt attribute is used for alternative text, descriptive text that explains what the image shows. This has various uses. An important one is that it provides a description of the image to screen reader users.

Table

Produces a table.

A range of elements are used.

  • <table> is a top-level element used to define the elements within the table.
  • <tr> is a second-level element used to define a row.
  • <td> elements are used to designate each cell within a row.
  • <th> is much like <td>, except that <th> is used specifically for column headings.

Form

(NB: My notes on this probably aren't as good as this resource.)

Produces a form for submission of data.

A range of elements are used.

  • <form> is a top-level element used to define the elements within the form. Also used to define the action that will occur when the form data is submitted (see submit below).
  • Most data input sections consists of a <label> element and a <input> element.
    • <label> states what this input section is for. It should include a for attribute that matches the id attribute of its associated <input> element.
    • <input> is a field for inputting the data. It should include a type attribute which specifies the kind of input (eg. text, a number, radio buttons), an id attribute that matches the for attribute of the associated <label> element, and a name attribute that provides an accessible name for screen readers.
  • <textarea> is another input field used for large text such as reviews and communications.
  • A <button> with its type attribute set to submit concludes the form. Clicking opn this will initiate the action specified in the <form> element, which will usually be processing and communication of the form data. Its value attribute specifies the text shown in the button.
  • Alternativelt an <input> element with its type attribute set to "submit can be used in the same way as the <button> element.

The default formatting of forms is often non-ideal. They have lots of quirks. These can be circumvented with CSS.

<form id="contact-form">

<label for="name">Name</label>
<input type="text" name="first-name" id="first-name">

<label for="email">Email</label>
<input type="text" name="email" id="email">

<label for="message">Message</label>
<textarea name="message" id="message"></textarea>

<input type="submit" value="Send" id="contact-form-submit">
</form>
